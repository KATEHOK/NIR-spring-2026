# Тесты серверной части (интеграционные)

Тесты проверяют корректность работы API сервера аутентификации: регистрацию,
вход, обновление токенов, выход, валидацию access-токена.
Все тесты выполняют реальные HTTP‑запросы к работающему серверу и записывают
время ответов в CSV‑файл.

## Структура

```
tests/
├── conftest.py                     # Фикстуры и хелперы
├── test_api.py                     # Интеграционные тесты API
├── requirements-test.txt           # Зависимости для тестов
├── results_api_integrations.csv    # Автоматически создаваемый файл с результатами
└── README.md                       # Этот файл
```

## Требования

- Работающий сервер аутентификации (FastAPI + PostgreSQL), доступный по
  `http://localhost:8000`.
- База данных проинициализирована (выполнена команда `python -m src.init_db`).

## Установка зависимостей

Из каталога `tests/`:

```bash
pip install -r requirements-test.txt
```

## Запуск

Из каталога `server/app/tests/`:

```bash
pytest test_api.py -v
```

После прогона в `results_api_integrations.csv` будут сохранены реальные времена
ответов для каждого проверенного эндпоинта.

## Собранные метрики

Файл `results.csv` содержит столбцы:

- `test_name` – имя тестовой функции
- `endpoint` – URL эндпоинта
- `status_code` – HTTP‑статус ответа
- `elapsed_ms` – время ответа в миллисекундах

## Примечания

- Тесты используют синхронный HTTP‑клиент (`httpx.Client`).
- Успешный `login-accept` не автоматизирован (требует вычисления
  корректного OTP), проверяются только неудачные сценарии (некорректный OTP, незарегистрированный пользователь и тд).
