# Прототип системы мультифакторной аутентификации

[Демонстрация](https://youtu.be/IajmY2cxHq0?si=mG-Ke_3tlPwk2WVR)

Данный проект представляет собой прототип системы мультифакторной аутентификации. Система состоит из компонентов:

- **Сервер аутентификации** (FastAPI) - обрабатывает регистрацию, вход, выпуск JWT и проверку OTP;
- **База данных** (PostgreSQL) - хранит пользовательские данные: id, пароль, refresh-токены, challenge для OTP и тд;
- **Мобильное приложение под Android** (OTP-токен) - генерирует одноразовые пароли на основе challenge с временной меткой.

Проект реализован как минимально рабочий прототип и предназначен для демонстрации основного функционала.

## Запуск сервера

### 1. Требования к окружению
- Операционная система: Linux (рекомендуется Ubuntu 22.04+) или любая ОС с Vagrant.
- Установленные инструменты: **Vagrant**, **VirtualBox**.
- Сервер использует порты: `8000` (API) и `5432` (PostgreSQL).

### 2. Подготовка окружения
Перейдите в каталог репозитория и выполните:

```bash
cd /path/to/repo
vagrant up           # запуск виртуальной машины (если используется Vagrant)
vagrant ssh          # вход в ВМ
cd /app              # внутри ВМ перейдите в рабочую директорию
```

### 3. Генерация секретов и ключей
Внутри ВМ в директории `/app/scripts` находятся два скрипта. Выполните их для создания файлов в папке `.secrets`:

```bash
cd /app
chmod +x scripts/*.sh
./scripts/gen_secrets.sh   # генерирует db_password и secret_key
./scripts/gen_certs.sh     # генерирует пару RSA-ключей для JWT
```

### 4. Запуск сервисов Docker

```bash
docker compose up -d
```

После запуска будут доступны:
- API сервер: `http://localhost:8000`
- Документация Swagger: `http://localhost:8000/docs`

### 5. Инициализация базы данных

Выполните скрипт инициализации внутри контейнера:

```bash
docker compose exec server python -m src.init_db
```

### 6. Переменные окружения

Файл `.env` в папке `/app` должен содержать следующие параметры (пример):

```env
# Server
SERVER_PORT=8000
SERVER_IP=0.0.0.0
SERVER_CORS=http://localhost:8000,http://127.0.0.1:8000
FAULT_LIMIT=3
FAULT_LIMIT_UPDATE_PERIOD=60
BAN_PERIOD=1440

# DB
DB_NAME=auth_db
DB_HOST=db
DB_PORT=5432
DB_USER=auth_user

# JWT
JWT_ALGORITHM=RS256
ACCESS_TOKEN_EXPIRE_MINUTES=5
REFRESH_TOKEN_EXPIRE_MINUTES=10
FAST_REFRESH_TOKEN_EXPIRE_MINUTES=5
```

### 7. Проверка работоспособности сервера

Откройте `http://localhost:8000/docs` в браузере. Вы увидите эндпоинты:

- `GET /auth/hello` – проверка работы сервера.
- `GET /auth/touch-db` – проверка соединения с БД.
- `GET /auth/public-key` – публичный ключ для верификации JWT.
- `POST /auth/register-init`, `POST /auth/register-accept` – двухэтапная регистрация OTP-токена.
- `POST /auth/login-init`, `POST /auth/login-accept` – двухэтапный вход.
- `POST /auth/refresh`, `DELETE /auth/logout` – управление токенами (обновление JWT и отзыв refresh).

Для успешного входа необходимо использовать мобильное приложение OTP-токен.

## Мобильное приложение OTP-токен (Android)

### Требования
- Устройство с **Android 7.0 (API 24) или выше**.
- Наличие **датчика отпечатков пальцев** (биометрия).
- Поддержка **Keystore** и **EncryptedSharedPreferences** (встроены в Android).

### Сборка и установка
1. Откройте проект `otp_token/OTPTokenApp` в **Android Studio**.
2. Подключите устройство по USB или Wi‑Fi для отладки.
3. Запустите приложение (Run → Run 'app').

### Первый запуск и регистрация
1. После установки приложение запросит отпечаток пальца (для первичной аутентификации).
2. На экране **Регистрация OTP-токена** будет сгенерирован **PIN-код** (строка из 8 символов). Скопируйте его.
3. Выполните регистрацию пользователя на сервере и в OTP-токене:
   - Отправьте `POST /auth/register-init` с `pin_code` и желаемым `password`. В ответ получите `key_part` и `refresh_token`.
   - В приложении нажмите **Далее (ввести key_part)**, введите полученный `key_part` и подтвердите регистрацию в приложении.
   - Отправив `POST /auth/register-accept` с полученным ранее refresh-токеном для подтверждения регистрации.
4. Приложение сохранит ключ пользователя в зашифрованном виде, после чего перейдёт на главный экран.

### Генерация OTP и вход
1. На сервере инициируйте вход: `POST /auth/login-init` с `user_id` и `password`. В ответе будет `challenge` (строка Base64).
2. Скопируйте `challenge` и вставьте его в поле **Введите challenge** в приложении.
3. Нажмите **Сгенерировать OTP**, подтвердите отпечаток. Приложение вычислит OTP.
4. Скопируйте OTP и отправьте его на сервер через `POST /auth/login-accept` (вместе с полученным ранее `refresh_token`).
5. При успешной проверке сервер вернёт `access_token` и новый `refresh_token`.
6. **После успешного входа нажмите в приложении кнопку «Подтвердить вход»** – это синхронизирует счётчик `login_count` на устройстве. Если этого не сделать, последующие OTP будут неверными.

### Дополнительные возможности
- **Сброс токена** – удаляет все данные приложения, необходимо заново пройти регистрацию.
- **Копирование OTP** – кнопка копирования появляется после генерации OTP.

## Примечания к прототипу

- Сервер не использует ротацию refresh-токенов, каждый refresh-токен действует до истечения срока или отзыва.
- Мобильное приложение фиксирует экран в портретной ориентации (для упрощения управления состоянием).
- Все зашифрованные данные (ключ пользователя, счётчики) хранятся в `EncryptedSharedPreferences`.

---

## Безопасность

- Мастер-ключ `EncryptedSharedPreferences` **не требует биометрии** для упрощения прототипа.
- JWT подписываются асимметрично (RS256). Приватный ключ хранится в `.secrets/jwt_private_key.pem` и игнорируется гитом.
